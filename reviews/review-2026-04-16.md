## 3. Accessibility (WCAG 2.1 / 2.2)

### 3.1 id="main-content" - Fixed

The `<main>` element now has `id="main-content"` set directly in HTML on all checked pages. The skip-nav link will work even if JavaScript fails. Good fix.

### 3.2 WCAG 2.2 Considerations

WCAG 2.2 is now the referenced standard for most accessibility regulations. Key new criteria to consider:

- **2.5.8 Target Size (Minimum)**: Interactive targets should be at least 24x24 CSS pixels. Review small icon buttons (terminal, ideas, news icons in the header) to ensure they meet this minimum. The `.icon-btn` class in the header may need padding adjustments.
- **2.4.11 Focus Not Obscured**: Ensure the keyboard focus indicator is not hidden behind sticky headers or overlapping elements. The site's header has `backdrop-filter` and `position: fixed` which could obscure focus on elements near the top of the page.
- **3.3.7 Redundant Entry**: Not currently relevant (no multi-step forms), but worth noting for future feature development.

### 3.3 Colour Contrast - Unchanged

The low-opacity white text issue flagged last week (`rgba(255,255,255,0.45)` on dark backgrounds) has not been addressed. This affects the hero subtitle, some meta text, and footer elements. These likely fall below the 4.5:1 contrast ratio requirement for normal-sized text.

**Recommended action:** Increase the opacity of `rgba(255,255,255,0.45)` to at least `rgba(255,255,255,0.65)` where used for body-sized text, or test with a contrast checker and adjust.
